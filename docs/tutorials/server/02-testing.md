# الدرس الثاني: اختبار جانب الخادم

## الهدف

فهم اختبار دوال الأخطاء وواجهات الـ API بطريقة عملية بدون خادم حقيقي.

## ملفات الاختبار المرتبطة

- `app/tests/apiErrors.test.ts`
- `app/tests/api.test.ts`

## أنماط الاختبار المستخدمة

- محاكاة `fetch` لطبقة client API.
- محاكاة `next/server` عند الحاجة.
- فحص status + payload في حالات النجاح والفشل.

## حالات مهمة مغطاة

- أخطاء المصادقة/الحدود/الخادم من OpenAI.
- غياب `OPENAI_API_KEY`.
- فشل الشبكة في دوال `app/lib/api.ts`.

## تشغيل سريع

```bash
npm run test
```

للتفاصيل الكاملة راجع `docs/testing.md`.
